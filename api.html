Getting Started

The Orbsurv API is a RESTful API that uses JSON for request and response payloads. All API requests should be made to the base URL below.

Base URL https://api.orbsurv.com/v1
Content Type application/json

Your First API Call

Here's a simple example to list all cameras in your account:

cURL 
curl -X GET https://api.orbsurv.com/v1/cameras \
  -H "Authorization: Bearer YOUR_API_KEY"

Authentication

All API requests require authentication using an API key. Include your API key in the Authorization header of every request.

Using Your API Key

Add your API key to the Authorization header:

Header 
Authorization: Bearer YOUR_API_KEY

You can generate and manage API keys from your account dashboard under Settings â†’ API Keys.

Cameras

Manage and control your Orbsurv cameras through the API. List cameras, retrieve details, and send control commands.

GET /v1/cameras

Retrieve a list of all cameras in your account.

Request 
curl -X GET https://api.orbsurv.com/v1/cameras \
  -H "Authorization: Bearer YOUR_API_KEY"
Response 
{
  "data": [
    {
      "id": "cam-001",
      "name": "Main Entrance",
      "status": "online",
      "position": 0.5,
      "created_at": "2024-01-15T10:30:00Z"
    }
  ],
  "meta": {
    "total": 1,
    "page": 1
  }
}
GET /v1/cameras/{id}

Retrieve detailed information for a specific camera.

Parameter Type Location Description
id * string path The unique identifier for the camera
Request 
curl -X GET https://api.orbsurv.com/v1/cameras/cam-001 \
  -H "Authorization: Bearer YOUR_API_KEY"
POST /v1/cameras/{id}/control

Send control commands to a camera (move, scan, patrol, etc.).

Parameter Type Required Description
action * string Yes Action to perform: move, scan, patrol, stop
position number No Position on track (0.0 to 1.0) for move action
Request 
curl -X POST https://api.orbsurv.com/v1/cameras/cam-001/control \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "action": "move",
    "position": 0.5
  }'

Feeds

Access live video feeds and recorded playback from your cameras.

GET /v1/feeds/{camera_id}/live

Get the live feed URL for a camera.

GET /v1/feeds/{camera_id}/playback

Retrieve recorded footage for a specific time range.

Events

Retrieve and manage events and alerts from your surveillance system.

GET /v1/events

Retrieve a list of events and alerts with optional filtering.

Webhooks

Receive real-time notifications when events occur in your Orbsurv system. Configure webhooks to integrate with your applications.

POST /v1/webhooks

Create a new webhook endpoint to receive event notifications.

Request 
curl -X POST https://api.orbsurv.com/v1/webhooks \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-server.com/webhook",
    "events": ["motion.detected", "alert.triggered"]
  }'

Rate Limits

API requests are rate-limited to ensure fair usage and system stability. Rate limits are applied per API key.

Standard Plan 100 requests/minute
Enterprise Plan 1,000 requests/minute

When you exceed the rate limit, you'll receive a 429 Too Many Requests response. The response includes Retry-After header indicating when you can retry.

Error Handling

The API uses standard HTTP status codes to indicate success or failure. All errors return a JSON response with details about what went wrong.

Status Code Description
200 Success - Request completed successfully
201 Created - Resource created successfully
400 Bad Request - Invalid request parameters
401 Unauthorized - Invalid or missing API key
404 Not Found - Resource not found
429 Rate Limit Exceeded - Too many requests
500 Server Error - Internal server error
Error Response Format 
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Request validation failed",
    "details": [
      {
        "field": "position",
        "message": "Position must be between 0 and 1"
      }
    ],
    "request_id": "req_123456789"
  }
}
