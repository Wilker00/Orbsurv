Playbook

Connect phones, rails, and backend services in minutes

Follow the steps below to bring a phone-based RTSP feed and the Arduino rail online. The checklist covers both LAN preparation and FastAPI configuration.

Quick status

  • FastAPI running
  • Phone stream reachable
  • Rail controller optional

Edit this card as you validate each piece.

1. Prepare the network

  1. Put the Android device and the dev laptop on the same Wi-Fi network.
  2. Disable battery optimisations for the RTSP app to avoid throttling.
  3. Reserve an IP (DHCP reservation) if you need a stable address when testing in the lab.

2. Configure the phone feed

  1. Install an RTSP server app (e.g. IP Webcam, RTSP Camera).
  2. Start the stream at rtsp://PHONE_IP:8554/live.
  3. Confirm you can open the URL in VLC on your laptop.

3. Launch FastAPI backend

  1. Download backend/models/yolov8s.onnx.
  2. Run pip install -r backend/requirements.txt.
  3. Export ORBSURV_RTSP=rtsp://PHONE_IP:8554/live.
  4. Start uvicorn app.main:app --reload.

4. Optional rail controller

  1. Flash Arduino firmware that understands RVEL:<float>\n.
  2. Connect USB and note the serial port (ls /dev/tty* or Device Manager).
  3. Export ORBSURV_SERIAL_PORT before launching uvicorn.
  4. Monitor feedback in the FastAPI console (look for RPOS packets).

Useful commands

# ping the phone
ping PHONE_IP
# sanity check RTSP with ffplay
ffplay rtsp://PHONE_IP:8554/live
# export counts for the admin view
python backend/admin.py list --pretty
# serial debug mode
python -m serial.tools.miniterm COM4 115200

LAN tip

Keep the phone on 5 GHz and disable sleep so the RTSP feed stays stable. Reboot the app if the stream stutters.
