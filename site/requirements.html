System Requirements

Hardware, network, and software requirements for Orbsurv deployment

Hardware Requirements

Orbsurv Camera Unit

  • Rail-mounted camera module
  • On-device AI processing
  • Thermal and motion sensors
  • PoE++ or Wi-Fi 6E connectivity
  • Optional 5G cellular module

Network Infrastructure

  • PoE++ switch (802.3bt) for PoE deployment
  • Gigabit Ethernet network
  • Wi-Fi 6E access points (for wireless)
  • Minimum 10 Mbps per camera
  • Recommended 25 Mbps for optimal performance

Installation Requirements

  • Rail mounting system (included)
  • Minimum 8-foot ceiling height
  • Power outlet (if not using PoE)
  • Network drop or Wi-Fi coverage
  • Professional installation recommended

Network Requirements

Bandwidth

Minimum
10 Mbps per camera for standard quality
Recommended
25 Mbps per camera for high-quality streaming
Cloud Storage
Additional bandwidth for cloud backup (optional)

Software Requirements

Web Browser

  • Chrome 90+ (recommended)
  • Firefox 88+
  • Safari 14+
  • Edge 90+
  • JavaScript enabled

Mobile App

  • iOS 14+ (iPhone/iPad)
  • Android 10+
  • Available on App Store and Google Play
  • Push notifications supported

Cloud Services

  • Orbsurv cloud account (required)
  • Optional cloud storage subscription
  • API access for integrations
  • Webhook support

Storage Requirements

On-Device Storage

Each Orbsurv camera includes onboard storage for local recording. Storage capacity depends on recording quality and retention settings.

Standard Quality
~30 days of continuous recording (default)
High Quality
~15 days of continuous recording
Event-Only Recording
Extended retention for motion/alert events

Integration Requirements

API Integration

  • RESTful API with JSON responses
  • API key authentication
  • Webhook support for real-time events
  • Rate limits: 100 requests/minute (standard), 1000 requests/minute (enterprise)
