Camera Prototype Admin Handbook

This page captures the important details the engineering team needs while we integrate the FastAPI backend, RTSP phone feed, rail controller, and WebSocket UI.

Architecture

  • FastAPI core at backend/app/main.py starts the vision task and exposes /ws + /health.
  • Vision loop vision.py — RTSP ingest, YOLOv8 ONNX inference, OC SORT tracker, controller step.
  • Rail controller control.py — proportional guidance + optional serial ASCII protocol.
  • Static UI under backend/static/index.html for quick overlays and state monitoring.

Service checklist

  1. Download models/yolov8s.onnx (see backend README).
  2. Install requirements and export ORBSURV_RTSP.
  3. Run uvicorn app.main:app --reload.
  4. Open http://localhost:8000/static/index.html for overlay smoke test.
  5. Optionally set ORBSURV_SERIAL_PORT once the rail firmware responds to RVEL.

Target policy

The backend always prefers humans:

  • When people are visible, choose the widest person track.
  • After 2 s without a person, fall back to vehicles (car, bus, truck, motorbike).
  • No targets for 3 s? Switch to Scan mode and issue a gentle sweep.

Default controller gains

These environmental variables are set in main.py and can be overridden as needed:

  • kp = 1.0
  • vmax = 0.35
  • deadband = 0.02
  • scan velocity = 0.08 (flips at ±0.65)

WebSocket Schema

{
  "kind": "tracks",
  "payload": {
    "tracks": [
      {"x": 0.42, "y": 0.3, "w": 0.2, "h": 0.4, "cls": "person", "conf": 0.82, "track_id": 7}
    ]
  }
}

Other kinds: state (mode, rail_pos, cmd_vel) and metrics (fps).

Serial reference

  • Outbound: RVEL:<float>\n
  • Inbound: RPOS:<float>\n and optional RERR:<code>\n
  • Firmware should respond at 10 Hz or faster for smooth motion.
